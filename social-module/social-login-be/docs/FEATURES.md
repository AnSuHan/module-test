# Features

`social-login-be`는 `ash-social-module`을 활용하여 다음과 같은 기능을 제공합니다.

## 1. 소셜 로그인 인증 흐름 지원

- **인가 URL 생성**: 각 소셜 공급자(Google, Kakao, Naver)의 OAuth2 인가 페이지로 리다이렉트하기 위한 URL을 생성합니다.
- **CSRF 방어**: 인가 요청 시 `state` 파라미터를 생성 및 검증하여 CSRF 공격을 방지합니다.
- **콜백 처리**: 소셜 공급자로부터 전달받은 `code`를 검증하고 사용자 정보를 획득합니다.
- **JWT 발급**: 인증 성공 시 사용자 정보를 포함한 자체 JWT(Access Token, Refresh Token)를 발급합니다.

## 2. 공급자 관리 및 영속성

- **멀티 공급자 지원**: 설정(`application.yml`)만으로 새로운 소셜 공급자를 활성화하거나 구성을 변경할 수 있습니다.
- **DB 영속성**: 공급자 설정 및 OAuth `state`를 PostgreSQL DB에 저장하여 다중 인스턴스 환경에서도 안정적인 세션 관리가 가능합니다.
- **자동 시드 로딩**: 애플리케이션 기동 시 설정된 공급자 정보를 DB에 자동으로 동기화(Upsert)합니다.

## 3. 보안 및 인프라

- **Spring Security 통합**: 모듈 엔드포인트에 대한 보안 설정을 자동으로 구성합니다.
- **환경 변수 관리**: `.env` 파일을 통해 민감한 자격 증명(Client ID, Secret)을 안전하게 관리합니다.
- **로그 추적**: `DEBUG` 레벨 로그 설정을 통해 소셜 로그인 흐름을 상세히 추적할 수 있습니다.
